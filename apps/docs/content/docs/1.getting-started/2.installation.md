Add @github-tools/sdk to this existing project for use with the Vercel AI SDK.

- Install @github-tools/sdk, ai, and zod; ensure GITHUB_TOKEN can be read from the environment
- Show a minimal example using createGithubTools with generateText or streamText
- Use a preset (e.g. code-review or repo-explorer) appropriate for this codebase unless full maintainer access is required
- Cite https://github-tools.com/getting-started/installation and https://github-tools.com/getting-started/quick-start

::

## Install the SDK

:::code-group
```bash [pnpm]
pnpm add @github-tools/sdk
```
```bash [npm]
npm install @github-tools/sdk
```
```bash [yarn]
yarn add @github-tools/sdk
```
```bash [bun]
bun add @github-tools/sdk
```
:::

## Pick your peer dependencies

What you install next depends on the [framework](/frameworks/ai-sdk) you build on:

| Framework | Import path | Peer dependencies |
|---|---|---|
| [AI SDK](/frameworks/ai-sdk) | `@github-tools/sdk` | `ai` (v6 or v7), `zod` |
| [eve](/frameworks/eve) | `@github-tools/sdk/eve` | `eve`, `ai` **v7**, `zod` |
| [Vercel Workflow](/frameworks/vercel-workflow) | `@github-tools/sdk/workflow` | `workflow`, `@ai-sdk/workflow`, `ai`, `zod` |
| [Chat SDK](/frameworks/chat-sdk) | `@github-tools/sdk` | `chat`, `@chat-adapter/github`, `ai`, `zod` |

### Required: AI SDK and Zod

Every entry point relies on [AI SDK](https://ai-sdk.dev/docs) for tool execution and [Zod](https://zod.dev) for schema validation:

:::code-group
```bash [pnpm]
pnpm add ai zod
```
```bash [npm]
npm install ai zod
```
```bash [yarn]
yarn add ai zod
```
```bash [bun]
bun add ai zod
```
:::

### Optional: eve agents

For [eve](https://eve.dev) filesystem-first agents, add `eve` and **`ai` v7** when you import from `@github-tools/sdk/eve`:

:::code-group
```bash [pnpm]
pnpm add eve ai
```
```bash [npm]
npm install eve ai
```
```bash [yarn]
yarn add eve ai
```
```bash [bun]
bun add eve ai
```
:::

See [Build a GitHub agent with eve](/frameworks/eve) — the full agent is 3 files.

### Optional: durable workflows

For **durable** agents with the [Vercel Workflow SDK](https://vercel.com/docs/workflow), add `workflow` and `@ai-sdk/workflow` when you import from `@github-tools/sdk/workflow` and use `createDurableGithubAgent`:

:::code-group
```bash [pnpm]
pnpm add workflow @ai-sdk/workflow
```
```bash [npm]
npm install workflow @ai-sdk/workflow
```
```bash [yarn]
yarn add workflow @ai-sdk/workflow
```
```bash [bun]
bun add workflow @ai-sdk/workflow
```
:::

See [Durable agents with Vercel Workflow](/frameworks/vercel-workflow) for the full pattern.

## Set your GitHub token

The SDK reads `GITHUB_TOKEN` from your environment automatically. Create a `.env` file at the root of your project:

```bash [.env]
GITHUB_TOKEN=github_pat_xxxxxxxxxxxx
```

You can also pass the token explicitly if you prefer:

```ts [explicit-token.ts]
createGithubTools({ token: 'github_pat_xxxxxxxxxxxx' })
```

::warning
Never commit tokens. Use `.env` locally and secret managers (Vercel, GitHub Actions secrets) in CI/production.
::

## Choose a token type

| Token type | When to use | Scope control |
|---|---|---|
| [Fine-grained PAT](https://github.com/settings/personal-access-tokens/new) | Production assistants | per-repository, per-permission |
| Classic PAT | Quick prototyping | broad, org-level |
| [Vercel Connect](/guide/tokens-and-auth#mint-tokens-with-vercel-connect) | Agents deployed on Vercel | per-installation, per-permission, short-lived |

See [Tokens and authentication](/guide/tokens-and-auth) for the detailed permission matrix and the Vercel Connect setup.

## Verify the setup

Run a minimal script to confirm the SDK initializes and a read tool resolves:

```ts [verify.ts]
import { createGithubTools } from '@github-tools/sdk'

const tools = createGithubTools({ preset: 'repo-explorer' })

console.log('Tools loaded:', Object.keys(tools).join(', '))
```

If the script prints tool names, your environment is ready. Continue with [Run your first call](/getting-started/quick-start).
